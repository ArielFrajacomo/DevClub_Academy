Node Structure

In this section, we will explore the structure of a Node.js project, including the organization of files and directories.

Project Structure

A typical Node.js project has the following structure:

my-node-project/
├── node_modules/       # Directory for installed dependencies
├── src/                # Source code for the application
│   ├── assets/         # Directory for static assets
│   ├── components/     # Directory for React components
│   │   ├── common/         # Directory for common React components 
│   │   ├── feature/        # Directory for feature-specific React components (feature-specific, like Hero, PricingTable)
│   │   ├── layout/         # Directory for layout React components (Header, Footer, Sidebar)
│   │   ├── ui/             # Directory for UI React components  (Button, Card, Input...)
│   ├── lib/            # Directory for React libraries
│   ├── pages/          # Directory for React pages
│   └── main.jsx        # Main entry point of the application
├── routes/             # Directory for route handlers
├── controllers/        # Directory for controller logic
├── models/             # Directory for database models
├── package.json        # Project metadata and dependencies
├── package-lock.json   # Lockfile for dependencies
└── README.md           # Project documentation

React Hooks

React Hooks are functions that let you use state and other React features in functional components. Some commonly used hooks include useState, useEffect, and useRef.

  • useState - Allows you to add state to functional components.
    Example: const [count, setCount] = useState(0);

    In this example, we are using the useState hook to create a state variable called count and a function called setCount to update that variable. The initial value of count is set to 0.

  • useEffect - Allows you to perform side effects in functional components.
    Example: useEffect(() => { console.log('Component mounted'); }, []);

    In this example, we are using the useEffect hook to log a message when the component mounts. The empty dependency array [] ensures this effect runs only once.

  • useRef - Allows you to create a reference to a DOM element or a mutable value that persists across renders.
    Example: const inputRef = useRef(null);

    In this example, we are using the useRef hook to create a reference to an input element. This reference can be used to access the DOM element directly.
